# anuraFIXED
A fixed version of the GitHub Codespaces setup script for [Anura OS](https://github.com/mercuryworkshop/anuraos) by [MercuryWorkshop](https://github.com/mercuryworkshop).

To set up everything, make a GitHub Codespace for this repository and type ```source start.sh```. That will set up everything you need and you can just continue building Anura OS normally with ```source codespace-basic-setup.sh``` and etc.

I also added some of my own features to the set up script. You can run ```export AUTO=1``` before running ```source codespace-basic-setup.sh``` to enable "Auto mode", which is a feature that makes it so that when you start the GitHub Codespace, the Anura OS server is automatically started as well. It also makes it so the GitHub Codespaces setup script automatically sets up the rootfs and starts the server after running ```make all```. You can also run ```export EXTREME=1``` before running the script to enable "Extreme mode", which is a feature that makes it so that Anura OS uses the GitHub codespaces directly (like a proxy) for terminal-based internet access. This feature is great to enable if ```cloudflare-dns.com``` is blocked somehow. Anura OS needs Cloudflare because it is the best and only way it can reliably connect to any website or server or etc. The only downside to enabling Extreme mode is that if you want to connect to the internet in Anura OS at all, you need the GitHub Codespace to be running.

I also made it so that the Anura OS server is automatically set to public port visibility because it is the only way internet access can work in Anura OS after you've installed it with offline support and Extreme mode enabled. However, the public port visibility setting is only enabled automatically when you have Extreme mode and Auto mode enabled.
